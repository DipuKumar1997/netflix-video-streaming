## Branch Information

This repository follows a structured branching strategy to separate live code, development work, and the project’s initial setup.

### master (default branch)
- This is the **default and live branch** of the repository.
- Contains all **stable and important code**.
- Represents the production-ready version of the project.

### feature/es-connection (development branch)
- Used as the **development environment**.
- All new features and ongoing development happen in this branch.
- Changes are merged into `master` once they are stable.

### init-project (initial branch)
- Contains the **initial starting version** of the project.
- Kept only for historical/reference purposes.
- Not used for active development.
